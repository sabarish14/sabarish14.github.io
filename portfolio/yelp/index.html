Description

This project is developed as a course project for Machine Learning course.This is a backend application which predicts the important factors that contribute to the rating of fast food restaurants.

The project was done with the following goals in mind:

  • To develop topic modeling algorithm to extract topics from Yelp reviews
  • To develop a prediction model that predicts the sentiment of a review according to the factors found in the previous step
  • To develop a regression model to predict the rating of a restaurant with the sentiment predicted in the previous step.

    • Reflections

    Preprocessing:

  • We extracted the reviews from the json file. Here as we are dealing with reviews of Restaurants for Arizona thus we have applied filtering at this stage to extract relevant reviews.
  • Now, we loop through all the reviews in the initial dataset and for each review, we split the review into sentences, remove stopwords, extract parts-of-speech tags for all the remaining tokens, filters out all words which are not nouns, uses Lemmatizer/Stemmer to lookup the lemma of each noun.
  • We have used just the Nouns as using other POS were not giving us good topics, that we are interested in. We have also used lemmatizer should as to strengthen words that belong to a single lemma.
    • Latent Dirchilet Allocation: Here, we have used Python’s LDA package to perform the LDA on the filtered collections that we have obtained above. We have basically three steps for this process:
  • Combine all the reviews pertaining to a business id
  • Feed these reviews as documents to the LDA and set the following parameters:
  • Number of Iterations:1500
  • Number of topics per document(business id ): 200
  • The output of this LDA algorithm is all the topics pertaining to a business id
    • Topic Selection: From the above section, we have obtained the latent topics for each of the business id.In this section, we explain our topic selection mechanism.We used the following steps to select the top 3 topics:
  • Count the frequencies of the topics from all the business ids.
  • Sort the vocabulary of topics in descending order to select the top 3 topics.
    • Classification
  • We build classifiers for each of the topics extracted by LDA, service, location and Price.
  • Training Data for this classification task is Human labelled reviews, containing the topic words and their relevant synonyms.
  • Size of the training data is 10000 reviews.
  • Building the classifier involved the below steps:
    • Feature Selection
  • Unigrams : We used unigram features to build this model, using nltk word tokenizer.
  • TF-IDF transformation: We used term frequency-inverse document frequency (tf-idf) for the features to consider only most frequent features ordered by tf-idf value.
    • Building Classification Model
  • The preprocessed data is used for feature extraction.
  • We used scikit learn python package to train our model. The features are given as input to classifiers.
  • We experimented on supervised classifiers such as Linear SVM (Support Vector Machines), Logistic Regression.
    • Support Vector Regression:
  • The labels predicted by the SVM classifier for the three topics is summed up and normalized for each business id.
  • These values serve as the features for regression. The rating of each restaurant is the expected output.
  • The Support Vector Regression is trained with the features and ratings. The kernel used for regression is RBF.
  • The accuracy of the prediction is evaluated using R square score.
